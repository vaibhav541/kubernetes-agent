Introduction

This is a demo application code where we mimic calculating optimal transaction costs and fetching trading data. These are high CPU and memory-based operations respectively, so when they are triggered, they will create a CPU spike and memory spike. Our agent will monitor them and try to make a quick remedy solution or optimize the algorithm via PR creating issues.

CPU Intensive Operation

This operation calculates optimal transaction costs across multiple trading paths, requiring significant CPU resources.

Memory Intensive Operation

This operation fetches and processes large volumes of trading data, consuming significant memory resources.
